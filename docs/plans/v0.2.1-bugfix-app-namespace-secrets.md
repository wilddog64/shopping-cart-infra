## Problem

All four app-layer services crash with authentication failures after `make up`:

- **basket-service**: `NOAUTH Authentication required` â€” connects to Redis without a password
- **order-service**: `secretRef: order-service-secrets` is a static secret with hardcoded `DB_PASSWORD: "changeme456"` â€” needs to come from Vault KV instead
- **product-catalog**: `secretRef: product-catalog-secrets` has hardcoded `DATABASE_PASSWORD: "changeme"` â€” does not match Vault KV value
- **frontend**: CrashLoopBackOff (separate investigation needed; no secret dependency identified)

**Root cause:** All ExternalSecrets created in PR #24 are in `shopping-cart-data` namespace. Kubernetes secrets are namespace-scoped â€” pods in `shopping-cart-apps` cannot mount secrets from `shopping-cart-data`. Each app namespace needs its own ExternalSecret syncing from the same Vault KV paths.

Specifically:
- `basket-service` expects `redis-cart-secret` with key `password` in `shopping-cart-apps`
- `order-service` mounts `order-service-secrets` (via `secretRef`) with keys `DB_USERNAME`, `DB_PASSWORD`, `RABBITMQ_USERNAME`, `RABBITMQ_PASSWORD` in `shopping-cart-apps`
- `product-catalog` mounts `product-catalog-secrets` (via `secretRef`) with keys `DATABASE_USER`, `DATABASE_PASSWORD`, `RABBITMQ_USER`, `RABBITMQ_PASSWORD` in `shopping-cart-apps`

## Fix

### File 1 â€” `data-layer/secrets/redis-cart-apps-externalsecret.yaml` (new)

```yaml
---
# ExternalSecret: redis-cart password synced into shopping-cart-apps
# basket-service expects secret named "redis-cart-secret" with key "password"
apiVersion: external-secrets.io/v1
kind: ExternalSecret
metadata:
  name: redis-cart-apps
  namespace: shopping-cart-apps
  labels:
    app.kubernetes.io/name: external-secret
    app.kubernetes.io/instance: redis-cart-apps
    app.kubernetes.io/component: cache-credentials
    app.kubernetes.io/part-of: shopping-cart
spec:
  refreshInterval: 24h
  secretStoreRef:
    name: vault-backend
    kind: ClusterSecretStore
  target:
    name: redis-cart-secret
    creationPolicy: Owner
    template:
      type: Opaque
      data:
        password: "{{ .password }}"
  data:
    - secretKey: password
      remoteRef:
        key: secret/data/redis/cart
        property: password
```
