# Bugfix: v0.2.1 — RabbitMQ credentials via Vault (addendum to PR #25)

**Branch (shopping-cart-infra):** `fix/app-namespace-secrets` — push directly to this branch; it is already open as PR #25
**Branch (k3d-manager):** `k3d-manager-v1.0.3` — push directly to this branch; it is already open as PR #60

---

## Problem

RabbitMQ StatefulSet runs with the built-in `guest/guest` default credentials — no
`RABBITMQ_DEFAULT_USER`/`RABBITMQ_DEFAULT_PASS` is set. The app-namespace ExternalSecrets
added in the earlier commit hardcode `RABBITMQ_USERNAME: "guest"` / `RABBITMQ_PASSWORD: "guest"`
in the template, bypassing Vault entirely.

---

## Fix

### Repo 1 — `shopping-cart-infra` (branch `fix/app-namespace-secrets`)

#### Change 1 — new file `data-layer/secrets/rabbitmq-externalsecret.yaml`

Creates `rabbitmq-credentials` secret in `shopping-cart-data` for the RabbitMQ StatefulSet.

```yaml
---
# ExternalSecret: RabbitMQ default credentials for the broker StatefulSet
# Syncs username + password from Vault KV into shopping-cart-data/rabbitmq-credentials
apiVersion: external-secrets.io/v1
kind: ExternalSecret
metadata:
  name: rabbitmq-credentials
  namespace: shopping-cart-data
  labels:
    app.kubernetes.io/name: external-secret
    app.kubernetes.io/instance: rabbitmq-credentials
    app.kubernetes.io/component: message-queue-credentials
    app.kubernetes.io/part-of: shopping-cart
spec:
  refreshInterval: 24h
  secretStoreRef:
    name: vault-backend
    kind: ClusterSecretStore
  target:
    name: rabbitmq-credentials
    creationPolicy: Owner
    template:
      type: Opaque
      metadata:
        labels:
          app.kubernetes.io/name: rabbitmq
          app.kubernetes.io/instance: rabbitmq-credentials
          app.kubernetes.io/component: message-queue-credentials
          app.kubernetes.io/part-of: shopping-cart
      data:
        username: "{{ .username }}"
        password: "{{ .password }}"
  data:
    - secretKey: username
      remoteRef:
        key: secret/data/rabbitmq/default
        property: username
    - secretKey: password
      remoteRef:
        key: secret/data/rabbitmq/default
        property: password
```

#### Change 2 — `data-layer/rabbitmq/statefulset.yaml`

Add two env vars reading from `rabbitmq-credentials` secret. Insert after the existing
`RABBITMQ_ERLANG_COOKIE` env var (line 74).

**Exact old block:**
```yaml
            - name: RABBITMQ_ERLANG_COOKIE
              value: "secretcookie"  # TODO: Replace with Secret in Stage 2
```

**Exact new block:**
```yaml
            - name: RABBITMQ_ERLANG_COOKIE
              value: "secretcookie"  # TODO: Replace with Secret in Stage 2
            - name: RABBITMQ_DEFAULT_USER
              valueFrom:
                secretKeyRef:
                  name: rabbitmq-credentials
                  key: username
            - name: RABBITMQ_DEFAULT_PASS
              valueFrom:
                secretKeyRef:
                  name: rabbitmq-credentials
                  key: password
```
