## Problem

Spring Boot AMQP auto-configuration creates a `CachingConnectionFactory` using
`spring.rabbitmq.*` properties. No `SPRING_RABBITMQ_*` env vars are set, so the factory
defaults to `localhost:5672`. Spring's built-in `RabbitHealthIndicator` uses this
auto-configured factory â€” every startup probe hits `localhost`, gets `Connection refused`,
returns 503, and the pod is killed.

**Root cause log lines:**
```
INFO  CachingConnectionFactory : Attempting to connect to: [localhost:5672]
WARN  RabbitMQHealthIndicator  : Health check failed
WARN  RabbitHealthIndicator    : Rabbit health check failed
```

The custom `ConnectionManager` connects correctly (reads `RABBITMQ_HOST` from ConfigMap),
but the health indicator does not use it.

## Changes

### Change 1 â€” `shopping-cart-order/k8s/base/configmap.yaml`

Add three lines under the `# RabbitMQ connection` block:

**Exact old block:**

```yaml
  # RabbitMQ connection
  RABBITMQ_HOST: "rabbitmq.shopping-cart-data.svc.cluster.local"
  RABBITMQ_PORT: "5672"
  RABBITMQ_VHOST: "/"
```

**Exact new block:**

```yaml
  # RabbitMQ connection
  RABBITMQ_HOST: "rabbitmq.shopping-cart-data.svc.cluster.local"
  RABBITMQ_PORT: "5672"
  RABBITMQ_VHOST: "/"
  SPRING_RABBITMQ_HOST: "rabbitmq.shopping-cart-data.svc.cluster.local"
  SPRING_RABBITMQ_PORT: "5672"
  SPRING_RABBITMQ_VIRTUAL_HOST: "/"
```

### Change 2 â€” `shopping-cart-infra/data-layer/secrets/postgres-orders-apps-externalsecret.yaml`

Add two lines to the `template.data` block so Spring AMQP auto-config picks up
Vault-managed credentials. The Vault source is the same as `RABBITMQ_USERNAME`/`RABBITMQ_PASSWORD`
â€” no new Vault paths needed.

**Exact old block:**

```yaml

        RABBITMQ_PASSWORD: "{{ .rabbitmq_password }}"
```

**Exact new block:**

```yaml
      data:
        DB_USERNAME: "{{ .username }}"
        DB_PASSWORD: "{{ .password }}"
        RABBITMQ_USERNAME: "{{ .rabbitmq_username }}"
        RABBITMQ_PASSWORD: "{{ .rabbitmq_password }}"

        SPRING_RABBITMQ_PASSWORD: "{{ .rabbitmq_password }}"
```
