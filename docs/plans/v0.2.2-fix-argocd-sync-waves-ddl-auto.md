# Fix: v0.2.2 — ArgoCD sync waves + ddl-auto=create for sandbox

**Branch:** `fix/argocd-sync-waves-ddl-auto`

---

## Problem

Two independent race conditions cause pods to crash on every fresh `make up`:

### Race 1 — StatefulSets start before ExternalSecrets sync

ArgoCD deploys all `data-layer` resources in one wave with no ordering. Kubernetes schedules
StatefulSet pods immediately. The pods read their credentials from Secrets — but those Secrets
are created by ESO's ExternalSecret controller, which needs Vault to respond and may take
10–30s to sync. Result: Redis/RabbitMQ pods start with `CHANGE_ME` or `CreateContainerConfigError`.

### Race 2 — Hibernate `validate` fails on empty or stale DB

`order-service` and `product-catalog` run with `spring.jpa.hibernate.ddl-auto: validate` on
the Kubernetes profile. On a fresh provision, no tables exist → validation fails at startup.
On re-provision of a long-running cluster, the image may have evolved (e.g. integer → UUID
for `order_items.id`) while the DB schema did not → validation fails with wrong column type.

---

## Fix

### Change 1 — `argocd/config/argocd-cm.yaml`

Add a custom health check so ArgoCD waits for each ExternalSecret to be `SecretSynced`
before advancing past wave 0. Insert before the closing of the `data:` block.

**Exact old block:**
```yaml
  resource.customizations.health.argoproj.io_Application: |
```

**Exact new block:**
```yaml
  resource.customizations.health.external-secrets.io_ExternalSecret: |
    hs = {}
    if obj.status ~= nil then
      for _, condition in ipairs(obj.status.conditions or {}) do
        if condition.type == "Ready" then
          if condition.status == "True" then
            hs.status = "Healthy"
            hs.message = condition.message or ""
          else
            hs.status = "Degraded"
            hs.message = condition.message or ""
          end
          return hs
        end
      end
    end
    hs.status = "Progressing"
    hs.message = "Waiting for ExternalSecret to sync"
    return hs

  resource.customizations.health.argoproj.io_Application: |
```

### Change 2 — All ExternalSecrets in `data-layer/secrets/`: add wave 0

Add `argocd.argoproj.io/sync-wave: "0"` to the `metadata.annotations` block of each
of the following 12 files. If `annotations:` does not exist under `metadata:`, add it.

Files:
- `data-layer/secrets/payment-encryption-externalsecret.yaml`
- `data-layer/secrets/payment-gateway-externalsecret.yaml`
- `data-layer/secrets/postgres-orders-apps-externalsecret.yaml`
- `data-layer/secrets/postgres-orders-externalsecret.yaml`
- `data-layer/secrets/postgres-payment-externalsecret.yaml`
- `data-layer/secrets/postgres-products-apps-externalsecret.yaml`
- `data-layer/secrets/postgres-products-externalsecret.yaml`
- `data-layer/secrets/rabbitmq-externalsecret.yaml`
- `data-layer/secrets/redis-cart-apps-externalsecret.yaml`
- `data-layer/secrets/redis-cart-externalsecret.yaml`
- `data-layer/secrets/redis-orders-cache-apps-externalsecret.yaml`
- `data-layer/secrets/redis-orders-cache-externalsecret.yaml`

**Pattern for each file — old:**
```yaml
metadata:
  name: <name>
  namespace: <namespace>
  labels:
```

**Pattern for each file — new:**
```yaml
metadata:
  name: <name>
  namespace: <namespace>
  annotations:
    argocd.argoproj.io/sync-wave: "0"
  labels:
```
