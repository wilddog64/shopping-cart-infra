# Spec: v0.3.0 — Manifest Cross-Check (pre-commit + CI)

**Branch:** `docs/next-improvements`
**Files:** `scripts/check-manifest-refs.sh`, `.pre-commit-config.yaml`, `.github/workflows/validate.yml`

---

## Problem

Every sandbox run surfaces new CrashLoopBackOff causes only visible after `make sync-apps`.
Root causes seen this milestone:

- `DATABASE_USER` in secret.yaml → app reads `DB_USERNAME` (key name mismatch)
- ExternalSecret keys didn't align with deployment `secretKeyRef` names
- Missing `RABBITMQ_USERNAME`/`RABBITMQ_PASSWORD` in base secret

The existing `validate.yml` catches YAML syntax and kustomize build errors but not
semantic mismatches between manifests. These should be caught **before commit**, not
after CI runs.

---

## What This Catches

- Key name mismatches between `secret.yaml` and deployment `secretKeyRef` ✅
- Key name mismatches between `configmap.yaml` and deployment `configMapKeyRef` ✅
- ExternalSecret `spec.data[].secretKey` values not present in base `secret.yaml` ✅

**Does NOT catch:** wrong values (semantic errors), runtime issues (NetworkPolicy, port binding).

---

## Approach

`scripts/check-manifest-refs.sh` runs as both a **pre-commit hook** (local, blocks commit)
and a **CI job** in `validate.yml` (enforces the same check on PRs). One script, two enforcement points.

## Change 1 — `scripts/check-manifest-refs.sh`

**Exact new file:**

```bash
#!/usr/bin/env bash
# scripts/check-manifest-refs.sh
#
# Cross-checks that all secretKeyRef/configMapKeyRef keys in Deployment manifests
# exist in the corresponding base secret.yaml / configmap.yaml, and that every
# ExternalSecret secretKey appears in the base secret.yaml for that secret name.
#
# Usage: bash scripts/check-manifest-refs.sh
# Exit: 0 = clean, 1 = mismatches found
set -euo pipefail

errors=0

# ── Require yq ────────────────────────────────────────────────────────────────
if ! command -v yq >/dev/null 2>&1; then
  echo "ERROR: yq is required (brew install yq)" >&2
  exit 1
fi

# ── Per-deployment checks ──────────────────────────────────────────────────────
while IFS= read -r deploy; do
  ns_dir=$(dirname "$deploy")

  # secretKeyRef: verify each referenced key exists in the base secret
  while IFS= read -r line; do
    secret_name=$(echo "$line" | yq '.secretKeyRef.name // ""')
    key=$(echo "$line"         | yq '.secretKeyRef.key  // ""')
    [[ -z "$secret_name" || -z "$key" ]] && continue

    secret_file=$(find "$ns_dir" -maxdepth 1 -name "secret.yaml" | head -1)
    if [[ -z "$secret_file" ]]; then
      # tolerate — base secret may live elsewhere (ESO-only repos)
      continue
    fi

    if ! yq '(.stringData // {}) | keys | .[]' "$secret_file" 2>/dev/null \
        | grep -qx "$key"; then
      echo "ERROR: $deploy — secretKeyRef key '$key' (secret: $secret_name) not in $secret_file" >&2
      errors=$((errors + 1))
    fi
  done < <(yq '.spec.template.spec.containers[].env[]?.valueFrom // {}' "$deploy" 2>/dev/null)

  # configMapKeyRef: verify each referenced key exists in the base configmap
  while IFS= read -r line; do
    cm_name=$(echo "$line" | yq '.configMapKeyRef.name // ""')
    key=$(echo "$line"     | yq '.configMapKeyRef.key  // ""')
    [[ -z "$cm_name" || -z "$key" ]] && continue

    cm_file=$(find "$ns_dir" -maxdepth 1 -name "configmap.yaml" | head -1)
    [[ -z "$cm_file" ]] && continue

    if ! yq '.data // {} | keys | .[]' "$cm_file" 2>/dev/null \
        | grep -qx "$key"; then
      echo "ERROR: $deploy — configMapKeyRef key '$key' (configmap: $cm_name) not in $cm_file" >&2
      errors=$((errors + 1))
    fi
  done < <(yq '.spec.template.spec.containers[].env[]?.valueFrom // {}' "$deploy" 2>/dev/null)

done < <(find . -path "*/k8s/base/deployment.yaml" -not -path "./.git/*")

# ── ExternalSecret checks ──────────────────────────────────────────────────────
while IFS= read -r es; do
  es_dir=$(dirname "$es")

  while IFS= read -r secret_key; do
    [[ -z "$secret_key" ]] && continue

    # ExternalSecret writes into the secret named in spec.target.name (or metadata.name)
    target=$(yq '.spec.target.name // .metadata.name' "$es" 2>/dev/null)

    secret_file=$(find "$es_dir" -maxdepth 1 -name "secret.yaml" | head -1)
    # Also search parent dir (infra repos keep secrets alongside data-layer)
    [[ -z "$secret_file" ]] && \
      secret_file=$(find "$(dirname "$es_dir")" -maxdepth 2 -name "secret.yaml" \
        | xargs grep -l "name: $target" 2>/dev/null | head -1 || true)
    [[ -z "$secret_file" ]] && continue

    if ! yq '(.stringData // {}) | keys | .[]' "$secret_file" 2>/dev/null \
        | grep -qx "$secret_key"; then
      echo "ERROR: $es — ExternalSecret secretKey '$secret_key' not in $secret_file" >&2
      errors=$((errors + 1))
    fi
  done < <(yq '.spec.data[].secretKey' "$es" 2>/dev/null)

done < <(find . -name "*externalsecret*.yaml" -not -path "./.git/*")

# ── Result ─────────────────────────────────────────────────────────────────────
if [[ $errors -gt 0 ]]; then
  echo "manifest-cross-check: $errors error(s)" >&2
  exit 1
fi
echo "manifest-cross-check: OK"
```

## Change 2 — `.pre-commit-config.yaml`

If the file does not exist, create it. If it exists, add the hook entry.

**Exact new file (or merge into existing):**

```yaml
repos:
  - repo: local
    hooks:
      - id: manifest-cross-check
        name: Manifest cross-check (secretKeyRef / configMapKeyRef / ExternalSecret)
        language: script
        entry: scripts/check-manifest-refs.sh
        pass_filenames: false
        files: \.(yaml|yml)$
```

## Change 3 — `.github/workflows/validate.yml` addition

Add this job after the existing `kustomize-build` job:

```yaml
  manifest-cross-check:
    name: Manifest Cross-Check
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install yq
        run: |
          curl -sSL https://github.com/mikefarah/yq/releases/download/v4.44.3/yq_linux_amd64 \
            -o /usr/local/bin/yq && chmod +x /usr/local/bin/yq

      - name: Run cross-reference check
        run: bash scripts/check-manifest-refs.sh
```

Also add `workflow_dispatch` to the `on:` triggers so the smoke-test job can be triggered manually:

```yaml
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  workflow_dispatch:
```

And add the smoke-test job (cluster-dependent, manual only):

```yaml
  smoke-test:
    name: Post-Sync Smoke Test
    runs-on: ubuntu-latest
    if: github.event_name == 'workflow_dispatch'
    steps:
      - uses: actions/checkout@v4

      - name: Install kubectl
        uses: azure/setup-kubectl@v4
        with:
          version: v1.32.0

      - name: Load kubeconfig
        run: |
          mkdir -p ~/.kube
          echo "${{ secrets.UBUNTU_K3S_KUBECONFIG }}" > ~/.kube/config
          chmod 600 ~/.kube/config

      - name: Wait for pods + dump crash logs
        run: |
          deadline=$((SECONDS + 300))
          while [[ $SECONDS -lt $deadline ]]; do
            bad=$(kubectl get pods -A --context ubuntu-k3s \
              --no-headers 2>/dev/null \
              | awk '$4 ~ /CrashLoopBackOff|Error/ {print $1"/"$2}')
            if [[ -z "$bad" ]]; then
              echo "All pods healthy"
              kubectl get pods -A --context ubuntu-k3s
              exit 0
            fi
            echo "Unhealthy pods: $bad — waiting 10s..."
            sleep 10
          done

          echo "## Crash logs" >> "$GITHUB_STEP_SUMMARY"
          printf '%s\n' "$bad" | while IFS='/' read -r ns pod; do
            echo "### $ns/$pod" >> "$GITHUB_STEP_SUMMARY"
            echo '```' >> "$GITHUB_STEP_SUMMARY"
            kubectl logs "$pod" -n "$ns" --context ubuntu-k3s --previous \
              --tail=50 2>&1 >> "$GITHUB_STEP_SUMMARY" || true
            echo '```' >> "$GITHUB_STEP_SUMMARY"
          done
          exit 1
```
