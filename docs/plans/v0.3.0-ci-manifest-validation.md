# Spec: v0.3.0 — CI Manifest Validation + Post-Sync Smoke Test

**Branch:** `docs/next-improvements`
**File:** `.github/workflows/validate.yml`

---

## Problem

Every sandbox run surfaces new CrashLoopBackOff causes that only become visible after
`make sync-apps`. Root causes seen this milestone:

- `DB_NAME: product_catalog` → wrong value, should be `products` (semantic ConfigMap error)
- `DATABASE_USER` in secret.yaml → app reads `DB_USERNAME` (key name mismatch)
- ExternalSecret keys didn't align with deployment `secretKeyRef` names
- NetworkPolicy blocked DNS (runtime; not statically detectable)
- nginx port mismatch — Dockerfile says 8080, old image had 80 (image staleness)

The existing `validate.yml` catches YAML syntax and kustomize build errors but not
semantic mismatches between manifests.

---

## Goal

Two additions to `validate.yml`:

### Job 1 — `manifest-cross-check` (cluster-free, runs on every PR)

A shell script (`scripts/check-manifest-refs.sh`) that:

1. **SecretKeyRef cross-check** — for each Deployment, find all `secretKeyRef.key` values;
   verify each key exists in the base `secret.yaml` for the referenced secret name.

2. **ConfigMapKeyRef cross-check** — same for `configMapKeyRef.key` vs `configmap.yaml`.

3. **ExternalSecret key alignment** — for each ExternalSecret, verify every
   `spec.data[].secretKey` appears as a key in the base `secret.yaml` for the same secret
   name (the base secret is the structural placeholder; ESO must produce the same keys).

Tool: `yq` (already available on GitHub Actions ubuntu runners via `snap`).

**Would have caught:**
- `DATABASE_USER` vs `DB_USERNAME` mismatch ✅
- Missing `RABBITMQ_USERNAME`/`RABBITMQ_PASSWORD` in base secret ✅

**Would NOT catch:**
- Wrong values (`product_catalog` vs `products`) — semantic, not structural
- Runtime issues (NetworkPolicy, port binding)

### Job 2 — `smoke-test` (manual trigger only, requires kubeconfig secret)

A workflow dispatch job (`workflow_dispatch`) that:

1. Installs kubectl
2. Loads kubeconfig from secret `UBUNTU_K3S_KUBECONFIG`
3. Waits up to 5 minutes polling `kubectl get pods -A --context ubuntu-k3s`
4. Any pod in `CrashLoopBackOff` or `Error` state → runs
   `kubectl logs <pod> -n <ns> --previous` and writes to job summary
5. Exits non-zero if any pod hasn't reached `Running` within timeout

**Trigger:** `workflow_dispatch` only (not on push/PR — cluster may not be up)

## Script Logic: `check-manifest-refs.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail

REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
errors=0

# Find all base deployments
while IFS= read -r deploy; do
  ns_dir=$(dirname "$deploy")

  # --- secretKeyRef checks ---
  while IFS= read -r ref; do
    secret_name=$(echo "$ref" | yq '.secretKeyRef.name')
    key=$(echo "$ref" | yq '.secretKeyRef.key')
    [[ "$secret_name" == "null" || "$key" == "null" ]] && continue

    secret_file=$(find "$ns_dir" -name "secret.yaml" | head -1)
    if [[ -z "$secret_file" ]]; then
      echo "WARN: $deploy refs secret/$secret_name but no secret.yaml in $ns_dir" >&2
      continue
    fi

    if ! yq ".stringData // {} | keys | .[]" "$secret_file" 2>/dev/null | grep -qx "$key"; then
      echo "ERROR: $deploy secretKeyRef key '$key' not found in $secret_file" >&2
      errors=$((errors + 1))
    fi
  done < <(yq '.spec.template.spec.containers[].env[]?.valueFrom' "$deploy" 2>/dev/null \
    | grep -A2 "secretKeyRef" || true)

  # --- configMapKeyRef checks ---
  while IFS= read -r ref; do
    cm_name=$(echo "$ref" | yq '.configMapKeyRef.name')
    key=$(echo "$ref" | yq '.configMapKeyRef.key')
    [[ "$cm_name" == "null" || "$key" == "null" ]] && continue

    cm_file=$(find "$ns_dir" -name "configmap.yaml" | head -1)
    [[ -z "$cm_file" ]] && continue

    if ! yq '.data // {} | keys | .[]' "$cm_file" 2>/dev/null | grep -qx "$key"; then
      echo "ERROR: $deploy configMapKeyRef key '$key' not found in $cm_file" >&2
      errors=$((errors + 1))
    fi
  done < <(yq '.spec.template.spec.containers[].env[]?.valueFrom' "$deploy" 2>/dev/null \
    | grep -A2 "configMapKeyRef" || true)

done < <(find . -path "*/k8s/base/deployment.yaml")

if [[ $errors -gt 0 ]]; then
  echo "manifest-cross-check: $errors error(s) found" >&2
  exit 1
fi
echo "manifest-cross-check: OK"
```

## `validate.yml` additions

```yaml
  manifest-cross-check:
    name: Manifest Cross-Check
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install yq
        run: |
          curl -sSL https://github.com/mikefarah/yq/releases/download/v4.44.3/yq_linux_amd64 \
            -o /usr/local/bin/yq && chmod +x /usr/local/bin/yq

      - name: Run cross-reference check
        run: bash scripts/check-manifest-refs.sh

  smoke-test:
    name: Post-Sync Smoke Test
    runs-on: ubuntu-latest
    if: github.event_name == 'workflow_dispatch'
    steps:
      - uses: actions/checkout@v4

      - name: Install kubectl
        uses: azure/setup-kubectl@v4
        with:
          version: v1.32.0

      - name: Load kubeconfig
        run: |
          mkdir -p ~/.kube
          echo "${{ secrets.UBUNTU_K3S_KUBECONFIG }}" > ~/.kube/config
          chmod 600 ~/.kube/config

      - name: Wait for pods + dump crash logs
        run: |
          deadline=$((SECONDS + 300))
          while [[ $SECONDS -lt $deadline ]]; do
            bad=$(kubectl get pods -A --context ubuntu-k3s \
              --no-headers 2>/dev/null \
              | awk '$4 ~ /CrashLoopBackOff|Error/ {print $1"/"$2}')
            if [[ -z "$bad" ]]; then
              echo "All pods healthy"
              kubectl get pods -A --context ubuntu-k3s
              exit 0
            fi
            echo "Unhealthy pods: $bad — waiting 10s..."
            sleep 10
          done

          echo "## Crash logs" >> "$GITHUB_STEP_SUMMARY"
          echo "$bad" | while IFS='/' read -r ns pod; do
            echo "### $ns/$pod" >> "$GITHUB_STEP_SUMMARY"
            echo '```' >> "$GITHUB_STEP_SUMMARY"
            kubectl logs "$pod" -n "$ns" --context ubuntu-k3s --previous \
              --tail=50 2>&1 >> "$GITHUB_STEP_SUMMARY" || true
            echo '```' >> "$GITHUB_STEP_SUMMARY"
          done
          exit 1
```
